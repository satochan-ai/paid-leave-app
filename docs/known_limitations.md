# 既知の制限事項

このアプリはlocalStorageを使ったプロトタイプです。以下の制限があります。

---

## データ保存・共有

| 制限 | 詳細 |
|------|------|
| 端末・ブラウザ依存 | localStorageはブラウザ・端末ごとに独立しているため、別端末・別ブラウザからはデータを参照できない |
| データ消失リスク | ブラウザのキャッシュ削除・localStorageクリアでデータが消える |
| 複数管理者の同時利用不可 | データ競合の仕組みがなく、複数端末から同時操作した場合の整合性は保証しない |
| サーバーDB保存なし | データはすべてクライアント（ブラウザ）側にのみ存在する |

---

## 認証・セキュリティ

| 制限 | 詳細 |
|------|------|
| パスワード平文保存 | パスワードはlocalStorageに平文で保存されており、開発者ツールで確認できる |
| パスワード平文表示 | 管理者画面の社員詳細に仮パスワードが平文表示される（プロトタイプ用仕様） |
| クライアントサイド認証のみ | 開発者ツールでlocalStorageを直接操作するとアクセス制御を回避できる |
| セッション管理なし | タブを閉じてもlocalStorageにセッション情報が残る |

---

## 業務ロジック

| 制限 | 詳細 |
|------|------|
| 出勤率8割判定未対応 | 労働基準法上の出勤率要件の判定は行っていない |
| 休職期間を考慮した厳密計算未対応 | 休職期間が出勤率・付与日数に影響するケースは未対応 |
| 申請内容編集未対応 | 申請後に内容を変更することはできない（取消→再申請で対応） |
| 承認済み申請の取消未対応 | 承認後の申請取消・有給復元には対応していない |
| メール通知未対応 | 失効予定アラートなどのメール通知機能はない |
| 時間単位有給未対応 | 時間単位の有給取得には対応していない（日単位・半日単位のみ） |
| 複数日まとめて申請未対応 | 有給申請は1件ずつの申請のみ対応 |
| CSVインポート未対応 | CSVからのデータ取り込みはできない |

---

## UI・操作

| 制限 | 詳細 |
|------|------|
| 操作履歴（Undo）なし | 削除・変更操作を元に戻す機能はない |
| 印刷・PDF出力未対応 | 画面の帳票出力は未対応 |
| Excel xlsx出力未対応 | CSV出力のみ対応（xlsxはなし） |
| 自動バックアップなし | 定期的な自動バックアップは未対応 |

---

## 未実装機能

| 機能 | 理由 |
|------|------|
| 本番認証 | JWTやセッション管理にはバックエンドが必要 |
| サーバーDB保存 | localStorageはクライアント限定 |
| メール通知 | サーバーサイド処理が必要 |
| 申請内容編集 | 取消→再申請による代替運用で対応 |
| 申請取消理由入力 | 簡易実装のため prompt() のみ対応（UIは未整備） |
| 承認済み申請の取消フロー | 業務上のルール整理が必要 |
| 出勤率8割判定 | 勤怠システムとの連携が必要 |
| 休職期間を考慮した厳密な付与判定 | 複雑な状態管理が必要 |
| 複数日まとめて申請 | 申請画面の拡張が必要 |
| 時間単位有給 | 労働基準法上の対応が複雑 |
| CSVインポート | データ検証・移行ロジックが複雑 |

---

## 起動方法に関する制限

`file://` で直接開いた場合、ブラウザのセキュリティ仕様によりlocalStorageや画面遷移が不安定になる可能性があります。

**症状例：**
- ログインができない
- 画面遷移後にデータが表示されない
- データが保存されない

**推奨：** ローカルサーバー（`python -m http.server 8080`）またはVS Code Live Serverで起動してください。  
詳細は `docs/local_setup_guide.md` を参照してください。

## GitHub Pages公開時の注意

GitHub Pagesで公開する場合、localStorageは閲覧者のブラウザごとに保存されます。
そのため、管理者と社員のデータがサーバーで共有されるわけではありません。

- 管理者が追加した社員データは、管理者のブラウザにのみ保存される
- 社員が別ブラウザでアクセスしても、そのデータは見えない
- デモ用途として使う場合は、同一ブラウザ・同一端末での確認に限定する
