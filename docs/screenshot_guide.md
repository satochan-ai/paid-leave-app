## 撮影前の共通準備

1. デモデータ初期化を実行してクリーンな状態にする
2. ブラウザのズームを 100% にする
3. ウィンドウ幅を 1280px 以上にする
4. 管理者アカウント（`admin@example.com` / `admin123`）でログイン済みにする

---

## 1. ログイン画面

| 項目 | 内容 |
|------|------|
| 撮影目的 | アプリの第一印象・デモ用ログイン情報の見やすさを示す |
| 見せたいポイント | デモ用ログイン情報カード（管理者・社員）・プロトタイプ注意書き |
| 撮影前の準備 | ログアウト状態にする |
| 注意点 | パスワードが平文表示されることを説明する場合は事前に断り書きを添える |

---

## 2. 管理者ダッシュボード（KPIカード）

| 項目 | 内容 |
|------|------|
| 撮影目的 | KPIカードで全社員の有給状況が把握できることを示す |
| 見せたいポイント | 社員数・在籍・休職・退職・有給残日数合計・失効予定の6カード |
| 撮影前の準備 | 管理者でログイン済み |
| 注意点 | KPIカードがすべて見えるように全体をキャプチャする |

---

## 3. クイックアクション

| 項目 | 内容 |
|------|------|
| 撮影目的 | 主要操作へのショートカットカードを示す |
| 見せたいポイント | 社員一覧・社員追加・有給付与チェック・CSV出力・バックアップの5カード |
| 撮影前の準備 | ダッシュボードを少し下スクロールしてクイックアクションを表示 |
| 注意点 | アイコン（絵文字）込みで5つのカードが見える状態で撮影 |

---

## 4. 社員一覧

| 項目 | 内容 |
|------|------|
| 撮影目的 | 全社員の有給状況が一覧で確認できることを示す |
| 見せたいポイント | 有給残日数列・ステータスバッジ・次回付与日・操作ボタン |
| 撮影前の準備 | 複数社員のデータが入った状態にする |
| 注意点 | テーブルが横スクロールできる状態を示す場合はスマホ幅で撮影 |

---

## 5. 社員詳細

| 項目 | 内容 |
|------|------|
| 撮影目的 | 社員個人の有給情報が詳細に確認できることを示す |
| 見せたいポイント | 基本情報・有給サマリーカード・付与履歴・取得履歴 |
| 撮影前の準備 | 付与履歴と取得履歴が1件以上ある社員を選ぶ |
| 注意点 | ログイン情報カードは別途撮影を推奨 |

---

## 6. 仮ログイン情報カード

| 項目 | 内容 |
|------|------|
| 撮影目的 | 管理者が社員のログイン情報を確認できることを示す |
| 見せたいポイント | ログインID・仮パスワード（等幅フォント）・権限・プロトタイプ警告文 |
| 撮影前の準備 | 社員詳細画面を開いてログイン情報カードが表示されている状態 |
| 注意点 | 「プロトタイプ用」であることを説明テキストと一緒に示す |

---

## 7. 有給取得登録

| 項目 | 内容 |
|------|------|
| 撮影目的 | 有給取得登録フォームの使いやすさを示す |
| 見せたいポイント | 対象社員情報カード（現在残日数・失効日）・日数選択・半休対応 |
| 撮影前の準備 | 社員詳細から「有給取得登録」へ遷移する |
| 注意点 | 半休（0.5日）の選択肢が表示されていることを確認する |

---

## 8. 有給付与チェック（実行結果）

| 項目 | 内容 |
|------|------|
| 撮影目的 | 付与漏れを自動検出する機能と実行結果を示す |
| 見せたいポイント | 「全社員の付与チェックを実行」ボタン・結果（チェック対象・新規付与件数・明細テーブル） |
| 撮影前の準備 | 入社日を1年以上前に設定した新規社員を追加してから実行する |
| 注意点 | 結果が「新たに付与する有給はありません。」の場合は新規社員を追加してテストする |

---

## 9. CSV出力セクション

| 項目 | 内容 |
|------|------|
| 撮影目的 | CSV出力機能を示す |
| 見せたいポイント | ダッシュボードのCSV出力セクション・3種のボタン |
| 撮影前の準備 | 管理者でログイン済み |
| 注意点 | ダウンロードされたCSVをExcelで開いた状態も合わせて撮影すると効果的 |

---

## 10. バックアップ/復元セクション

| 項目 | 内容 |
|------|------|
| 撮影目的 | データの外部保存・復元機能と危険操作の分離を示す |
| 見せたいポイント | バックアップダウンロードボタン・ファイル選択・復元ボタン・⚠ 危険操作セクション |
| 撮影前の準備 | ダッシュボードの「データ管理」および「⚠ 危険操作」セクションを表示 |
| 注意点 | データ管理と危険操作が明確に分かれていることを強調する |

---

## 11. 社員マイページ

| 項目 | 内容 |
|------|------|
| 撮影目的 | 社員が自分の有給情報だけ見られることを示す |
| 見せたいポイント | 大きな有給残日数・氏名・メタ情報・サマリーカード（次回付与日・失効日） |
| 撮影前の準備 | 社員アカウント（`yamada@example.com`）でログインする |
| 注意点 | 管理者向けの操作ボタン・CSV・バックアップが表示されていないことを確認する |
