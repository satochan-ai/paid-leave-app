# 有給休暇管理アプリ ポートフォリオ概要

## 1. 開発目的

社員ごとの有給付与・取得・残日数を管理できる、HTML/CSS/JavaScriptベースの有給休暇管理アプリを作成した。

バックエンド・DBなしで動作し、デモ・プロトタイプとして社内提案や技術検証に使える形を目指した。

---

## 2. 想定課題

| 課題 | 状況 |
|------|------|
| 有給残日数管理が属人的 | Excelや口頭での管理が多い |
| 付与日数・失効日の把握が難しい | 法定付与日数の計算が手作業 |
| 社員本人が残日数を確認しにくい | 管理者に都度聞かなければならない |
| データを外部保存・共有したい | CSVやバックアップが取れない |

---

## 3. 解決できること

- 管理者が全社員の有給状況を一覧確認できる
- 社員が自分の有給残日数をいつでも確認・申請できる
- 法定付与日数（通常付与・比例付与）に基づく自動管理
- 2年失効・古い付与分から消化に対応
- 有給申請→承認フローでペーパーレス化に対応
- CSV出力で社労士・管理部への共有が容易
- JSONバックアップ/復元でデータ移行・退避が可能
- 有給付与チェックで付与漏れを自動検出・補完

---

## 4. 技術構成

| 項目 | 内容 |
|------|------|
| HTML | 画面構造 |
| CSS | スタイリング（レスポンシブ対応） |
| JavaScript | ロジック・DOM操作・ルーティング |
| localStorage | データ永続化 |
| バックエンド | なし |

---

## 5. 実装済み機能

| 機能 | 詳細 |
|------|------|
| 認証 | ログイン・ログアウト・権限制御（管理者/社員） |
| 社員管理 | 追加・編集・一覧・詳細・退職扱い |
| 有給管理（通常付与） | 法定付与日数・2年失効・古い付与から消化・半休0.5日 |
| 有給管理（比例付与） | 週所定労働日数・時間・年間日数による比例付与計算 |
| 有給申請フロー | 申請→承認/却下・申請取消（pending のみ）・申請履歴 |
| CSV出力 | 社員一覧・有給残日数・取得履歴・社員別詳細・有給申請履歴 |
| バックアップ/復元 | JSONでlocalStorageデータを保存・復元（旧バックアップとの互換性あり） |
| 付与チェック | 入社日から付与漏れを自動検出・補完（全社員/社員別） |
| 社員マイページ | 残日数・勤務条件・次回付与日・申請フォーム・申請履歴 |
| UI | レスポンシブ対応・クイックアクション・危険操作分離 |

---

## 6. 工夫した点

- `storage.js` にlocalStorage操作を集約し、将来のDB移行コストを最小化
- `leaveCalculator.js` に計算ロジックを分離し、テスタブルな設計
- `routerGuard.js` でURLベースのアクセス制御を実装
- XSS対策として `esc()` 関数でHTML出力を全エスケープ
- デモデータ初期化と通常データ管理を明確に分離（危険操作セクション）
- CSV出力はBOM付きUTF-8でExcel文字化けを防止

---

## 7. 本番化時に必要な対応

| 対応項目 | 優先度 |
|---------|--------|
| Supabase/Firebase等によるDB移行 | 高 |
| 本番認証（JWTなど） | 高 |
| パスワードハッシュ化 | 高 |
| メール通知（申請・承認・失効アラート） | 中 |
| 申請内容編集・複数日申請 | 中 |
| 出勤率8割判定 | 低 |

## 公開時の位置づけ

このアプリは、有給休暇管理業務を想定したMVPプロトタイプです。

本番利用を目的としたものではなく、以下を示すためのポートフォリオ成果物です。

| 示したいこと | 実装内容 |
|------------|---------|
| 要件定義・画面設計 | `docs/` 配下に15ファイル以上の仕様書を整備 |

| 権限制御 | 管理者/社員の2ロール、URL ベースのアクセス制御 |

| バックアップ/復元 | JSON形式でのデータ永続化 |
| UI/UX | レスポンシブ対応、危険操作の分離、クイックアクション |
| ドキュメント整備 | デモシナリオ・QAレポート・ポートフォリオ資料まで作成 |
